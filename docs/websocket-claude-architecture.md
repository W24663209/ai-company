## 核心组件说明

### 1. 前端 WebSocket 管理器 (`app.js`)

```javascript
// 连接配置
const maxReconnectAttempts = 10;  // 最大重连次数
const pingInterval = 25000;       // 保活心跳间隔 (25秒)
const connectionTimeout = 10000;  // 连接超时 (10秒)

// 重连策略: 指数退避
const delay = Math.min(1000 * Math.pow(2, reconnectAttempts - 1), 5000);
// 第1次: 1秒, 第2次: 2秒, 第3次: 4秒, 第4次+: 5秒
```

**状态显示**:
- 未连接 (灰色)
- 连接中... (橙色)
- 已连接 (绿色)
- 重连中 (N/10)... (深橙色)
- 已断开 (红色)

### 2. 后端 WebSocket Handler (`agents.py`)

```python
@router.websocket("/ws/chat/{project_id}/{requirement_id}")
async def chat_ws(websocket: WebSocket, project_id: str, requirement_id: str):
    await websocket.accept()
    
    # 双任务并发:
    # 1. keepalive - 每30秒发送ping保活
    # 2. receive_messages - 接收消息并流式处理
    
    async for chunk in claude_service.chat_stream(...):
        # 流式推送给前端
        await websocket.send_json({"type": "partial", "data": ...})
```

### 3. Claude 服务 (`claude_service.py`)

```python
async def chat_stream(project_id, requirement_id, message):
    # 1. 加载已有会话
    sessions = _load_sessions(project_id)
    existing_session = sessions.get(requirement_id)
    
    # 2. 构建命令
    cmd = [
        settings.claude_bin,
        "-p", "--print",
        "--output-format", "json",  # JSON格式获取token消耗
        "--permission-mode", "bypassPermissions",
        "--resume", session_id      # 恢复会话避免重复历史
    ]
    
    # 3. 异步流式读取
    async for chunk in _async_stream_claude(cmd, prompt, cwd, env):
        yield chunk
```

## 容错机制

### 1. 网络断线自动恢复
- WebSocket 断开 → 自动重连 (最多10次)
- 指数退避策略避免服务器压力
- 重连期间显示状态提示

### 2. HTTP Fallback
- WebSocket 完全失败时，回退到 SSE 流式 HTTP 请求
- 保证消息能够送达

### 3. 会话恢复
- 使用 `--resume session_id` 继续对话
- 避免重复发送历史消息导致 token 浪费
- 会话 ID 持久化到文件系统

### 4. 超时处理
- 连接超时: 10秒
- 消息接收超时: 60秒 (触发ping检测)
- Claude 处理超时: 300秒 (由 ThreadPoolExecutor 控制)

## 性能优化

1. **requestAnimationFrame**: 前端使用 RAF 批量更新 DOM，避免页面卡顿
2. **ThreadPoolExecutor**: 后端使用线程池运行同步的 Claude 进程，不阻塞事件循环
3. **JSON 输出**: 使用 `--output-format json` 直接获取结构化数据，无需解析文本
4. **缓存命中**: Claude API 的 prompt caching 减少重复输入的 token 消耗

## 安全考虑

1. **SSH 密钥隔离**: 每个项目可使用不同的 SSH 密钥
2. **权限控制**: `--permission-mode bypassPermissions` 仅用于自动化场景
3. **路径限制**: Claude 只能访问项目目录和共享目录
4. **输入验证**: 后端对消息内容进行 JSON 解析验证

## 监控指标

- **连接成功率**: WebSocket 首次连接成功比例
- **重连次数分布**: 自动重连几次后成功
- **Token 消耗**: 按项目和需求统计 API 调用成本
- **响应时间**: 从发送到首字返回的延迟
- **错误率**: 各类错误的发生频率
